WANDER:
That tree is wearing last October's clothes. The botanical name is *marcescence*: the abscission layer at the base of the leaf — the small zone of cells a tree normally grows specifically in order to *drop something* — never formed, so the dead leaf is technically still attached to a living branch, sealed in place by a seal that didn't seal. Theories vary (deer-deterrence, nutrient timing so the litter falls when the soil can use it, a juvenile trait that fades with maturity), but what holds regardless is that the leaf will only come off when spring's new growth physically pushes it off. This is a different shape than a vestige. A vestige is a part whose function ended but whose form persists; marcescence is a part whose *removal was scheduled and didn't fire*. Software has both, and the marcescent kind is the more dangerous of the two — the deprecated API stamped "removed in v3" still riding along in v9, the temporary feature flag that's now load-bearing, the Y2K windowing pivots that quietly pushed the problem out to 2049 and 2069. There's a small bleak theory in here that organizations can't reliably issue release signals — abscission turns out to be hard, harder than addition — and that the only thing which actually clears dead matter is new growth so insistent that it breaks the seal from the other side. The leaf doesn't fall in autumn; it falls in May, when the twig swells. Probably true of most kinds of letting go.

BRANCH:
The asymmetry between *clearing* (passive, via replacement) and *retiring* (active, via decision) — when does a system actually possess the latter as a real capability rather than just claiming to.
